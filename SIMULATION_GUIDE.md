# Running the SUMO Simulation

This guide explains how to run the SUMO traffic simulation and generate the dataset for training the sensor fusion neural network.

---

## 1. Prerequisites

### 1.1 Install SUMO (Simulation of Urban Mobility)

SUMO is a free, open-source traffic simulation platform. You must install it before running the simulation.

#### Windows (Using Chocolatey)

```bash
choco install sumo
```

#### Windows (Manual Download)

1. Download from: http://sumo.dlr.de/download
2. Extract to a folder (e.g., `C:\SUMO\`)
3. Add to PATH or set `SUMO_HOME` manually

#### macOS

```bash
brew install sumo
```

#### Linux (Ubuntu/Debian)

```bash
sudo apt-get update
sudo apt-get install sumo sumo-tools sumo-doc
```

#### Verify Installation

```bash
sumo --version
```

You should see output like: `Version 1.16.0` (or newer)

---

### 1.2 Set SUMO_HOME Environment Variable

The Python scripts need to know where SUMO is installed. You must set the `SUMO_HOME` environment variable.

#### Windows PowerShell (Temporary - Current Session Only)

```powershell
$env:SUMO_HOME = "C:\Program Files\SUMO"
```

#### Windows PowerShell (Permanent - All Future Sessions)

```powershell
[Environment]::SetEnvironmentVariable("SUMO_HOME", "C:\Program Files\SUMO", "User")
```

Then restart PowerShell or VS Code.

#### Windows Command Prompt (Temporary)

```cmd
set SUMO_HOME=C:\Program Files\SUMO
```

#### Linux/macOS (Temporary - Current Session)

```bash
export SUMO_HOME=/usr/share/sumo
```

#### Linux/macOS (Permanent - Add to .bashrc or .zshrc)

```bash
echo 'export SUMO_HOME=/usr/share/sumo' >> ~/.bashrc
source ~/.bashrc
```

**Note**: Replace the path with your actual SUMO installation directory.

**How to find your SUMO_HOME path:**

```bash
# Try this command
which sumo
# or
sumo --help | head -1
```

Look for the directory that contains `bin/sumo` or `tools/`.

## 2. Folder Setup & Files Required

Before running the simulation, ensure these files exist in your project folder:

```
SUMO_Project/
├── simulation.sumocfg          ✅ MUST EXIST (SUMO config file)
├── grid.net.xml                ✅ MUST EXIST (network topology)
├── routes.rou.xml              ✅ MUST EXIST (vehicle routes)
├── trips.trips.xml             ⚠️  Optional (for regenerating routes)
├── fix_routes.py               ✅ MUST EXIST (preprocessing script)
├── generate_dataset_multi_vehicle.py    ✅ MUST EXIST (main script)
│
└── routes_fixed.rou.xml        ⏳ GENERATED by fix_routes.py
```

**Why each file matters:**

| File | Purpose | Auto-Generated? |
|------|---------|---|
| `simulation.sumocfg` | Tells SUMO which network and routes to load | No, pre-configured |
| `grid.net.xml` | 3×3 grid road network (9 intersections) | No, pre-configured |
| `routes.rou.xml` | 100+ vehicle routes without type info | No, pre-configured |
| `routes_fixed.rou.xml` | Same routes but with vehicle types added | **Yes, by fix_routes.py** |
| `fix_routes.py` | Adds vehicle type definitions to routes | No, provided script |
| `generate_dataset_multi_vehicle.py` | Main simulation & data generation | No, provided script |

## 3. Step-by-Step Instructions to Run

### Step 1: Verify Files Are Present

Before starting, confirm all required files exist:

```bash
# Windows PowerShell
Test-Path simulation.sumocfg
Test-Path grid.net.xml
Test-Path routes.rou.xml
Test-Path fix_routes.py
Test-Path generate_dataset_multi_vehicle.py
```

All should return `True`.

### Step 2: Set SUMO_HOME (if not already set)

```powershell
# Test if SUMO_HOME is set
$env:SUMO_HOME

# If empty, set it:
$env:SUMO_HOME = "C:\Program Files\SUMO"

# Verify
sumo --version
```

### Step 3: Prepare Routes (Add Vehicle Types)

The `routes.rou.xml` file lacks vehicle type definitions. The `fix_routes.py` script adds these:

```bash
python fix_routes.py
```

**Expected output:**
```
✅ routes_fixed.rou.xml created
```

This creates `routes_fixed.rou.xml` with:
- **Slow vehicles** (30%): maxSpeed=12 m/s, accel=1.0, σ=1.0
- **Normal vehicles** (50%): maxSpeed=18 m/s, accel=2.5, σ=0.7
- **Fast vehicles** (20%): maxSpeed=30 m/s, accel=4.0, σ=0.4

**Troubleshooting:**
- If error: `FileNotFoundError: routes.rou.xml`, make sure `routes.rou.xml` exists in the folder
- If script runs but no output file, check file permissions

### Step 4: Run the Main Simulation

Now run the data generation script:

```bash
python generate_dataset_multi_vehicle.py
```

**What happens:**

1. SUMO starts and loads the simulation
2. 100+ vehicles begin driving on the 3×3 grid network
3. Script samples interactions between vehicles
4. Data is written row-by-row to `dataset_final.csv`
5. Progress updates appear every 50 simulation steps

**Expected console output:**

```
Step 50 | Rows: 1245
Step 100 | Rows: 2489
Step 150 | Rows: 3734
...
Step 850 | Rows: 50000
```

**Runtime:** ~5-15 minutes depending on your CPU

**Progress indicators:**

| Parameter | Value |
|-----------|-------|
| Target rows | 50,000 |
| Max simulation steps | 4,000 |
| Vehicles in simulation | 100+ |
| Typical time to complete | 5-15 minutes |

## 4. Output Explanation

### 4.1 Generated File: dataset_final.csv

After the simulation completes, you'll have `dataset_final.csv` with 50,000 rows of clean, ground-truth data.

### 4.2 Dataset Columns

| Column | Type | Unit | Range | Meaning |
|--------|------|------|-------|---------|
| `time` | integer | steps | 0-300 | Simulation timestep |
| `ego_id` | string | - | 0-99 | Subject vehicle ID |
| `target_id` | string | - | 0-99 | Nearby vehicle ID |
| `distance` | float | meters | 0-80 | Distance between vehicles |
| `relative_speed` | float | m/s | -20 to 20 | Speed difference (ego - target) |
| `camera` | float | confidence | 0-1 | Vision system confidence |
| `ultrasonic` | integer | binary | 0 or 1 | Proximity alarm (active/inactive) |
| `angle` | float | radians | -π to π | Bearing to target vehicle |
| `risk_score` | float | probability | 0-1 | Ground-truth collision risk |

### 4.3 Column Explanations

**time**: Simulation timestep (0-300 seconds)
- Each simulation step represents ~0.075 seconds of real time

**ego_id / target_id**: Vehicle identifiers
- Example: `vehicle.0`, `vehicle.1`, etc.
- The script randomly samples 6 ego vehicles and up to 20 targets per step

**distance**: Actual distance between vehicle centers (meters)
- Calculated from 2D positions using Euclidean distance
- Only samples recorded if distance < 80m (realistic interaction range)

**relative_speed**: Speed difference (m/s)
- `ego_speed - target_speed`
- Negative = ego is slower (target catching up)
- Positive = ego is faster (moving away)

**camera**: Vision system confidence (0-1)
- Calculated as: `1 / (1 + 0.02 * distance)`
- 0.0 = completely obscured
- 1.0 = clear line of sight
- At 80m: ~0.38 confidence
- At 10m: ~0.83 confidence

**ultrasonic**: Proximity alarm (0 or 1)
- **1** = alarm active (target within 5-10m range)
- **0** = no alert
- Binary decision (unlike camera's continuous confidence)

**angle**: Bearing to target (radians, -π to π)
- Calculated using `atan2(Δy, Δx)`
- -π = target directly behind
- 0 = target directly ahead
- π/2 = target to the left
- -π/2 = target to the right

**risk_score**: Ground-truth collision risk (0-1)
- Formula:
  ```
  risk = 0.45 * (1/(distance+1)) 
       + 0.40 * (relative_speed/20) 
       + 0.10 * (1 - camera) 
       + 0.05 * ultrasonic
  ```
- Weighted factors:
  - Distance (45%): Most critical
  - Speed (40%): Time-to-collision
  - Camera failure (10%): Vision reliability
  - Ultrasonic (5%): Backup alert
- Range: 0.0 (safe) to 1.0 (imminent collision)

### 4.4 Dataset Sample Rows

```csv
time,ego_id,target_id,distance,relative_speed,camera,ultrasonic,angle,risk_score
15,vehicle.5,vehicle.12,45.67,3.2,0.578,0,1.234,0.187
15,vehicle.5,vehicle.8,8.23,12.5,0.757,1,0.456,0.721
15,vehicle.5,vehicle.19,62.1,1.8,0.392,0,-2.845,0.156
...
```

## 5. Common Errors and Fixes

### Error 1: SUMO_HOME Not Set

**Error Message:**
```
Traceback (most recent call last):
  File "generate_dataset_multi_vehicle.py", line 12, in <module>
    sys.exit("Please declare SUMO_HOME")
```

**Cause:** The `SUMO_HOME` environment variable is not set.

**Fix:**

```powershell
# Test if it's set
$env:SUMO_HOME

# If empty, set it:
$env:SUMO_HOME = "C:\Program Files\SUMO"

# Verify SUMO works
sumo --version
```

If this doesn't work, try finding SUMO manually:

```powershell
where sumo
# Note the directory, then set SUMO_HOME to the parent of the bin folder
```

### Error 2: ModuleNotFoundError: No module named 'traci'

**Error Message:**
```
ModuleNotFoundError: No module named 'traci'
```

**Cause:** The `traci` module is part of SUMO's Python tools, and the path is not set correctly.

**Fixes to try (in order):**

1. **Restart your terminal/VS Code** after setting SUMO_HOME
2. **Verify SUMO_HOME is set:**
   ```powershell
   $env:SUMO_HOME
   # Should output: C:\Program Files\SUMO (or your path)
   ```
3. **Check if SUMO tools exist:**
   ```powershell
   Test-Path "$env:SUMO_HOME\tools\traci.py"
   # Should output: True
   ```
4. **If SUMO_HOME path is wrong**, update it and restart

### Error 3: sumo: command not found

**Error Message:**
```
Cannot find SUMO executable
```

**Cause:** SUMO is not installed or not in PATH.

**Fixes:**

1. **Verify SUMO is installed:**
   ```bash
   sumo --version
   ```
   If this fails, reinstall SUMO (see Section 1.1)

2. **Add SUMO to PATH:**
   - Windows: Set `SUMO_HOME` to the installation directory
   - Linux/Mac: Check if `/usr/share/sumo` exists

### Error 4: FileNotFoundError: simulation.sumocfg

**Error Message:**
```
FileNotFoundError: [Errno 2] No such file or directory: 'simulation.sumocfg'
```

**Cause:** The script is not in the same folder as the SUMO config files.

**Fix:** Make sure you're in the correct directory:

```bash
# Verify you're in the SUMO_Project folder
pwd  # or 'cd' to show current directory

# List files to confirm they exist
ls simulation.sumocfg grid.net.xml routes_fixed.rou.xml
```

### Error 5: routes_fixed.rou.xml not found

**Error Message:**
```
FileNotFoundError during SUMO initialization
```

**Cause:** You didn't run `fix_routes.py` first.

**Fix:**

```bash
python fix_routes.py
# This creates routes_fixed.rou.xml
```

Then try running the simulation again.

### Error 6: Simulation Starts But Generates Few Rows

**Symptoms:**
- Script runs but generates only 100-1000 rows before stopping
- Takes less than 1 minute

**Cause:** Possible reasons:
1. Not enough vehicles in simulation
2. Vehicles getting stuck or removed
3. SUMO configuration issue

**Diagnostics:**

```python
# Add this line in generate_dataset_multi_vehicle.py after "traci.simulationStep()"
# to see vehicle count:
print(f"Step {step}: {len(traci.vehicle.getIDList())} vehicles")
```

**Fixes:**
1. Check `routes_fixed.rou.xml` has enough vehicles
2. Check `grid.net.xml` network is valid
3. Try rerunning `fix_routes.py`

### Error 7: SUMO GUI Opens Instead of Running Headless

**Symptom:** A SUMO window opens; simulation is slow

**Cause:** The config file specifies `--gui` mode

**Fix:** Edit `simulation.sumocfg`:

```xml
<!-- Change this: -->
<configuration>
    <gui-settings>
        <gui-only value="true"/>
    </gui-settings>
    ...
</configuration>

<!-- To this: -->
<configuration>
    <input>
        <net-file value="grid.net.xml"/>
        <route-files value="routes_fixed.rou.xml"/>
    </input>
    ...
</configuration>
```

Or directly modify the sumo_cmd in the Python script:

```python
sumo_cmd = ["sumo", "-c", "simulation.sumocfg"]  # Use 'sumo', not 'sumo-gui'
```

## 6. Customization Notes

### 6.1 Changing the Number of Rows (Dataset Size)

Edit `generate_dataset_multi_vehicle.py`:

```python
TARGET_ROWS = 50000  # Change this number
MAX_STEPS = 4000     # Simulation will stop after this many steps or reaching TARGET_ROWS
```

**Example:** To generate 100,000 rows:

```python
TARGET_ROWS = 100000
MAX_STEPS = 8000  # Increase max steps proportionally
```

**Tradeoff:**
- Larger dataset = longer runtime, more memory, better model training
- Smaller dataset = faster testing, less storage
- Typical: 50,000 rows takes 5-15 minutes

### 6.2 Adjusting Sensor Parameters

Edit `generate_dataset_multi_vehicle.py` to modify how sensors behave:

#### Camera Confidence Decay

```python
def camera_conf(d):
    return max(0.25, 1 / (1 + 0.02 * d))
    #                         ^ Change this to adjust distance effect
    #          ^ Minimum confidence value
```

- Increase `0.02` → more aggressive decay at distance
- Decrease `0.02` → more forgiving at distance
- Increase `0.25` → higher minimum confidence

#### Ultrasonic Detection Range

```python
ultra = 1 if d < random.uniform(5, 10) else 0
#                         ^    ^  Change these values
```

- `5, 10` = ultrasonic activates between 5-10 meters
- `10, 15` = extended range
- `2, 5` = shorter, more precise range

#### Risk Score Weights

```python
risk = (
    0.45*(1/(d+1)) +      # Distance weight (currently 45%)
    0.4*(rel_speed/20) +  # Speed weight (currently 40%)
    0.1*(1 - cam) +       # Camera weight (currently 10%)
    0.05*ultra            # Ultrasonic weight (currently 5%)
)
```

Adjust the `0.45, 0.4, 0.1, 0.05` values (must sum to 1.0)

Example: If distance is less important:
```python
risk = (
    0.35*(1/(d+1)) +      # Reduce distance
    0.5*(rel_speed/20) +  # Increase speed importance
    0.1*(1 - cam) +
    0.05*ultra
)
```

#### Speed Randomization

```python
speed = random.uniform(5, 25)  # Change these bounds
#                         ^  ^
#                       min max
```

- `5, 25` = vehicles drive 5-25 m/s (18-90 km/h)
- `10, 30` = slightly higher speeds
- `3, 15` = slower traffic

### 6.3 Adjusting Number of Ego/Target Vehicles Sampled

Edit `generate_dataset_multi_vehicle.py`:

```python
egos = random.sample(vehicles, min(6, len(vehicles)))
#                                   ^ Change this

for v in targets:
    targets = random.sample(
        [v for v in vehicles if v != ego],
        min(20, len(vehicles)-1)
        #   ^ Change this
    )
```

- **egos**: Number of subject vehicles to sample per step (currently 6)
  - Higher = more interaction diversity, slower
  - Lower = fewer samples, faster
  
- **targets**: Number of nearby vehicles per ego (currently 20)
  - Higher = more data samples per step
  - Lower = less data per step

### 6.4 Changing the Network or Routes

If you want to use a different road network or vehicle routes:

1. **Generate new network:**
   ```bash
   netgenerate --grid.number=5 --output-file=new_grid.net.xml
   # Creates a 5×5 grid instead of 3×3
   ```

2. **Update simulation.sumocfg:**
   ```xml
   <net-file value="new_grid.net.xml"/>
   ```

3. **Generate routes for new network:**
   ```bash
   duarouter --net-file=new_grid.net.xml \
             --trip-files=trips.trips.xml \
             --output-file=new_routes.rou.xml
   ```

4. **Prepare routes:**
   ```bash
   # Update fix_routes.py to use new_routes.rou.xml
   # Then run: python fix_routes.py
   ```

## 7. Performance Tips

### Speed Up Simulation

1. **Reduce target rows:**
   ```python
   TARGET_ROWS = 10000  # Test quickly
   ```

2. **Run headless (no GUI):**
   - Ensure `sumo_cmd = ["sumo", "-c", ...]` (not `sumo-gui`)

3. **Use fewer samples per step:**
   ```python
   egos = random.sample(vehicles, min(3, len(vehicles)))  # Down from 6
   targets = random.sample(..., min(10, len(vehicles)-1))  # Down from 20
   ```

4. **Disable verbose SUMO output:**
   ```xml
   <!-- In simulation.sumocfg: -->
   <report>
       <verbose value="false"/>  <!-- Change to false -->
   </report>
   ```

### Improve Data Quality

1. **Ensure enough vehicles:**
   - Check routes file has 100+ vehicles
   - Confirm grid network is complex enough

2. **Extend max steps:**
   ```python
   MAX_STEPS = 6000  # Let simulation run longer
   ```

3. **Increase interaction range:**
   ```python
   if d > 100:  # Instead of 80
       continue
   ```

## 9. Troubleshooting Checklist

Before reporting issues, verify:

- [ ] SUMO is installed: `sumo --version`
- [ ] SUMO_HOME is set: `$env:SUMO_HOME`

- [ ] Python dependencies installed: `pip list | grep -E "pandas|numpy|keras"`
